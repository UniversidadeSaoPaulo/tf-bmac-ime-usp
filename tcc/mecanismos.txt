Mecanismos de Arquitetura

Introdução

Este documento tem por objetivo detalhar as tecnologias (linguagens, ferramentas e frameworks) e metodologias (padrões de projeto) utilizadas na implementação do software, bem como as motivações para a escolha de cada uma delas.

Um pré-requisito importante é que todas as ferramentas utilizadas sejam gratuitas, e, sempre que possível, de código aberto. Isto garante que a ferramenta resultante possa ser ampliada e modificada por qualquer entidade interessada sem que hajam restrições de qualquer natureza.

Linguagem de Programação

É necessário fazer duas escolhas neste tópico: a da linguagem a ser utilizada para construir a ferramenta, e a da linguagem através da qual os algoritmos serão descritos para a mesma.

Para a aplicação, é preciso ter uma linguagem que ofereça performance, escalabilidade, suporte ao desenvolvimento para a web e facilidade para trabalhar com bancos de dados relacionais.

O algoritmo exige uma linguagem que possa ser compilada ou interpretada dinâmicamente, que tenha respaldo acadêmico (para aproveitar código pré-existente) e, preferencialmente, que já possua um ambiente de compilação/runtime livre (as complexidades inerentes a este tipo de implementação fogem ao escopo deste trabalho).
